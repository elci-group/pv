## Architectures & assumptions

```
A cached-nonstream  refire = events + 30x 120 s-heartbeats (coalesced), one
                   non-streaming call, atomic panel swap on completion
B streaming         refire every 8 s, tokens displayed as they arrive
accuracy            quality x tier-fit (penalty 15%/tier below requirement)
pass threshold      accuracy >= 40 + 8*(req tier - 1) — errors compound
                    across agents, bigger fleets need sharper advice
stability           A: 0.96 - churn(events) - 0.3*variance (cache dedupes)
                    B: 0.78 - variance - 0.04 flicker
staleness           A: min(1800/events, 60)s   B: interval/2 + ttft
cba = 0.45*acc + 0.25*stability + 0.15*freshness + 0.15*cost (ref $0.50/day)
```

## Verdict

CACHED NON-STREAMING IS STABLE ENOUGH TO COMPETE — and wins on cost.
Event-driven refires keep staleness under every workload's freshness
need while costing 9.9x less on average, with ~0.21 higher display
stability (no token flicker, no rephrase churn). Streaming only pays
when decisions must react in <8 s — agentic loops act in seconds,
so they never enter that regime.
